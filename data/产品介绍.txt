一、产品定位
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
本产品面向券商投研团队、公募/私募基金经理及个人量化研究者，
聚焦 A 股上市公司业绩预告场景，通过自动化爬取、文本解析与信号
量化，在传统财报发布前 1-3 周内，从公开非结构化文本中提取"业绩
超预期"的早期预警信号，辅助投资决策，降低人工阅读成本。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
二、核心功能
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 全量业绩公告实时抓取
   - 自动从东方财富网公有 API（RPT_PUBLIC_OP_NEWPREDICT 报表端点）
     抓取全部 A 股上市公司当日业绩预告流水数据。
   - 支持自动翻页，一次运行可获取全量当日公告（500条/页，直至最后一页）。
   - 抓取结果直接落库，支持增量更新，不产生重复记录（按股票代码去重）。

2. NLP 信号量化评分
   - 读取每条公告中的业绩增速预测区间（增幅下限/上限）。
   - 动态拉取行业真实的“市场一致预期”作为参考基准进行超预期差值对比，
     系统默认挂载 `akshare` 抓取真实机构预测值，遇到API限流或断网环境
     则向下兼容 fallback 至内置行业标准均值（如计算机25%，制造12%）。
   - 依据差值大小，自动将信号划分为四档：
       强信号（strong）：超出基准 ≥15%，评分 75-100
       中信号（medium）：超出基准 5%-15%，评分 50-74
       弱信号（weak）：超出基准 0%-5%，评分 30-49
       负面/无信号（negative）：未达基准，评分 0-29

3. 信号列表看板（Web 界面）
   - 展示近 30 天内发布公告且有量化信号的公司列表。
   - 默认按"业绩公告发布时间"由近到远排序；同日期再按信号评分由高到低。
   - 支持按"信号强度"进行筛选。
   - 点击任意行可进入该公司的信号详情页。

4. 信号详情页（Web 界面）
   - 展示公司基本面：股票代码、名称、所属板块。
   - 实时估值注入：在查看详情时，后端 `realtime_data.py` 将自动通过 
     Baidu/Akshare 接口实时拉取最新的 PE(TTM)、PB、PEG 回显给用户。
   - 实时业绩对比：拉取前序财报的真实“历史同期业绩”并展现。如第三方源
     暂无数据覆盖（如新股），则前端将触发智能降级自适应排版。
   - 展现公告原文及 AI推理依据。嵌入 ECharts 可视化图表。

5. 每日定时自动更新
   - 提供 fetch_real_data.py 脚本，可配合 Windows 任务计划程序实现
     每日自动抓取并更新数据，无需人工干预。

三、系统实现逻辑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

整体架构为标准三层结构：

  [数据采集层] ──▶ [后端 API 层] ──▶ [前端展示层]
       │                  │
  fetch_real_data.py   Django + DRF
  (Python + requests)  SQLite3 数据库
                       信号计算逻辑

步骤详解：

  Step 1  爬取
    fetch_real_data.py 使用 requests 库，以 GET 方式访问东方财富
    数据中心 API，循环翻页，汇总全量数据记录（List[Dict]）。

  Step 2  清洗与标准化
    - 行业：对 API 返回的 TRADE_MARKET_ZJH 字段进行关键词正则映射，
      归类到"计算机/电子/医药/新能源/金融/机械制造/汽车/基础化工/农林
      牧渔/食品饮料/综合"等标准行业。
    - 板块：依据股票代码前缀判断：60/00 前缀→主板，30 前缀→创业板，
      68 前缀→科创板。

  Step 3  NLP 信号计算
    - 从 ADD_AMP_LOWER、ADD_AMP_UPPER 字段提取增速区间。
    - 计算中枢：mid = (lower + upper) / 2
    - 通过 update_market_expectations 模块对接真实市场预期，
    - 计算差值：diff = mid - 市场真实一致预期基准
    - 依差值分档赋值 strength 和 score。

  Step 4  持久化存储
    - Company（公司）、FinancialText（公告文本）、Signal（量化信号）、
      Valuation（估值快照）四张表写入 SQLite3 数据库。
    - 通过 get_or_create 保证公司记录不重复。

  Step 5  API 服务
    Django REST Framework 提供 RESTful JSON API，
    支持分页（默认每页 20 条）、信号强度过滤，以及近 30 天自动过滤逻辑。

  Step 6  前端展示
    Vue 3 + Vite 单页应用，通过 Axios 调用后端 API，
    使用 Element Plus 组件库渲染表格/筛选器，
    使用 ECharts 渲染业绩比较柱状图。

四、所有数据字段定义与来源
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【Company 公司表】
  字段名         数据类型    含义                来源
  stock_code     文本        A 股股票代码（6位）  东方财富 SECURITY_CODE
  name           文本        公司简称             东方财富 SECURITY_NAME_ABBR
  industry       文本        所属证监会行业分类   东方财富 TRADE_MARKET_ZJH（清洗映射后）
  board          文本        上市板块             股票代码前缀规则推断

【FinancialText 公告文本表】
  字段名         数据类型    含义                来源
  source_type    文本        文本类型（业绩预告） 固定值 announcement
  publish_date   日期        公告发布日期         东方财富 NOTICE_DATE
  title          文本        公告标题             系统自动拼接
  content        文本        公告原文内容         东方财富 PREDICT_CONTENT
  processed      布尔        是否已解析           系统内部标志位

【Signal 信号表】
  字段名              数据类型    含义                      来源
  score               整数        超预期信号综合评分(0-100)  NLP算法计算
  strength            文本        信号强度（强/中/弱/负面）  NLP算法分类
  reasoning           文本        评分依据（自然语言）       系统自动生成
  expected_growth     文本        文本提取的预期增速中枢     东方财富 ADD_AMP_LOWER/UPPER中值
  market_expectation  文本        系统设定的市场基准增速     akshare获取真实值 / 后备行业固定均值
  historical_data     文本        历史同期增速快照           实时API拉取注入 / 占位回退
  generate_time       日期时间    信号入库时间               系统自动记录

【Valuation 估值表】
  字段名    数据类型    含义        来源
  pe        浮点数      市盈率PE    API动态拉取覆盖（底层存储用作模拟默认值 25.5 占位）
  pb        浮点数      市净率PB    API动态拉取覆盖（底层存储用作模拟默认值 3.2 占位）
  peg       浮点数      PEG         API动态拉取覆盖（底层存储用作模拟默认值 1.2 占位）

五、应用场景
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 报告季前的提前布局
   在每年的1月/4月/7月/10月业绩预告密集发布期，每日运行抓取脚本，
   可在财报正式披露前提前锁定"强信号"个股，形成超前研究视角。

2. 超预期事件驱动选股
   将信号评分 ≥75 的公司纳入下一日的重点跟池，结合股价走势回测
   信号有效性，构建超预期驱动的量化选股策略。

3. 降低投研人员阅读成本
   每日业绩预告可多达数百条，本产品自动评分后，投研人员只需
   关注评分前20名的公司，大幅压缩阅读时间。

4. 信号有效性验证
   可导出历史信号数据（CSV），结合外部行情数据，回测"强信号"公司
   在公告后 5/10/20 个交易日的超额收益情况。
